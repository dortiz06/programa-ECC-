¿Qué es ECC?
-------------
ECC (Elliptic Curve Cryptography) es criptografía asimétrica basada en curvas elípticas.
En este proyecto se utiliza la curva SECP256R1 (NIST P-256).

En ECC existen dos claves:
- Clave privada: debe mantenerse secreta.
- Clave pública: puede compartirse (por ejemplo, con otros usuarios o sistemas).


¿Cómo funciona en la app?
-------------------------
La app “Flipobook” simula el registro de usuarios y enseña cómo dos personas podrían
acordar una clave compartida de forma segura:

1. Al registrarse, el usuario genera su par de claves ECC (privada + pública).
2. Se simula otro usuario que también genera sus claves.
3. Con ECDH se calcula un secreto compartido a partir de:
   - la clave privada de un lado
   - la clave pública del otro lado
4. Ese secreto compartido se deriva a una clave simétrica mediante HKDF con SHA-256.
5. Con la clave derivada se cifra un mensaje de ejemplo para ilustrar confidencialidad.

Nota educativa: por ser un proyecto de aprendizaje, el servidor imprime material criptográfico
en consola para que puedas ver el flujo.


Derivación y cifrado (ECDH + HKDF + cifrado autenticado)
---------------------------------------------------------
El flujo criptográfico del módulo `crypto_ecc.py` es:

1. ECDH (Elliptic Curve Diffie-Hellman):
   - Produce un secreto crudo (bytes) que ambos extremos obtienen igual.

2. HKDF con SHA-256:
   - Convierte el secreto crudo en una clave simétrica lista para cifrado.
   - En este caso se derivan 32 bytes.

3. Cifrado con AES-256-GCM:
   - Usa la clave derivada para cifrar un mensaje.
   - AES-GCM además aporta autenticación (protege contra modificaciones).


Validaciones de seguridad
--------------------------
En `app.py` se valida todo en el backend antes de ejecutar ECC:

1. Nombre:
   - No vacío (ni solo espacios).

2. Correo:
   - Se valida con expresiones regulares (regex) para formato usuario@dominio.tld.

3. Celular:
   - Exactamente 10 dígitos (se eliminan caracteres no numéricos y se comprueba longitud).

4. Contraseña:
   - Mínimo 10 caracteres.
   - Al menos una mayúscula.
   - Al menos un número.
   - Al menos un carácter especial.


Aplicación en la vida real
----------------------------
En un sistema real, una red social podría usar ECC para:
- Establecer claves entre usuarios (acuerdo de claves tipo ECDH).
- Proteger mensajes privados (cifrado de extremo a extremo, por ejemplo).
- Minimizar tamaños de claves frente a alternativas como RSA.

Este proyecto es educativo y simula el “flujo completo” en memoria. En producción, además
se deberían implementar prácticas como:
- No exponer claves privadas.
- Usar intercambio efímero por sesión/mensaje.
- Almacenar y rotar claves de manera segura.
- Diseñar el protocolo completo con autenticación adecuada.
